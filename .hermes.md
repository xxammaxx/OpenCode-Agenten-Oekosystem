# Hermes Project Handoff

## Purpose

This project bundles portable Hermes handoff assets for the OpenCode bootstrap.

**Note on Hermes:** Hermes is a separate, Python-based tool (repository: `NousResearch/hermes-agent`), not embedded in OpenCode. Files in the `.hermes/` directory are **bootstrapping assets for Hermes**, not Hermes-native runtime files. They serve as a portable bundle source that Hermes can load when pointed at this repository.

## Instructions

- OpenCode is the primary coding executor.
- Hermes is the orchestrator and skill runtime — a Python agent, not an OpenCode built-in.
- Keep MCP servers disabled until explicitly reviewed.
- Prefer the project-local bundle assets instead of rewriting global Hermes config.
- Use the YAML skill bundle at `.hermes/skill-bundles/canonical-working-method.yaml` for the canonical workflow.

- no secret material
- no automatic remote CI
- no autonomous production writes
- no global config overwrite

## Known Limitations

- Hermes runtime live test is classified as `TOOL_GAP_HERMES_RUNTIME` — the Python-based Hermes agent is not installed by this bootstrap. Only the handoff assets are provided.
